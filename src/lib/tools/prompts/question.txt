Use this tool when you need to ask the user questions during execution. This lets you:
1. Gather user preferences or requirements
2. Clarify ambiguous instructions
3. Get decisions on implementation choices as you work
4. Offer the user a choice of directions to take

Usage notes:
- When custom is enabled (the default), a "type your own answer" field is added automatically; do not add an "Other" or catch-all option yourself.
- options is a list of {label, description}; with no options the user gets a free-text field.
- Set multiple: true to let the user select more than one option.
- If you recommend a specific option, make it the first in the list and append " (Recommended)" to its label.
- Write the questions and options in the user's language.
