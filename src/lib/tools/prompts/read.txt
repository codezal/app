Read a text file from the workspace, returned with line numbers for file:line references.

Usage:
- The path is relative to the workspace root.
- By default this returns up to 2000 lines from the start of the file.
- The offset parameter is the 1-based line number to start from. To read later sections, call this tool again with a larger offset — the footer reports the next offset to use.
- Use the grep tool to find specific content in large files or files with long lines, and the glob tool to look up filenames by pattern.
- Each line is prefixed by its line number in cat -n style (the line number, a tab, then the content). When copying text into edit_file, never include the line-number prefix.
- Any line longer than 2000 characters is truncated.
- Call this tool in parallel when you know there are multiple files you want to read.
- Avoid tiny repeated slices (e.g. 30-line chunks). If you need more context, read a wider window.
- Image files are returned as a base64 data URL; binary files return a short notice instead of their contents.
