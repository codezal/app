# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for a vulnerability.

Report security issues through GitHub private vulnerability reporting for this
repository. Include:

- Affected version or commit.
- Impact and affected feature area.
- Reproduction steps or proof of concept.
- Any known workaround.

We will acknowledge valid reports, investigate the issue, and coordinate a fix
before public disclosure when appropriate.

## Scope

Security reports are especially useful for:

- Provider credential handling.
- Local file access and workspace permissions.
- Terminal, process, and PTY execution.
- Plugin, MCP, hook, and tool execution paths.
- Update signing and release verification.

Official signed builds and updater artifacts are produced only by the Codezal
release infrastructure.
